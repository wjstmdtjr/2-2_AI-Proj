# Week 10 HW04 Design

## Goal

Complete the Week 10 HW04 integrated assignment by comparing model choices for image and text inputs. The submission should show that different input structures require different model architectures.

## Scope

Create an `Assignment4` folder containing executable code, a submission notebook, generated figures/results, and a short report. The work will be done on branch `assignment4-week10-hw04`, then merged back into `main` after verification.

## Source Basis

Use the professor-provided TensorFlow/Keras notebooks as the implementation style reference:

- Week09 MNIST CNN notebook for image classification.
- Week10 Korean sentiment notebook for text classification.

This keeps the assignment aligned with class material and avoids unnecessary framework changes.

## Part A: Image Task

Use MNIST because it is fast, stable, and directly supported by the Week09 class notebook.

Models:

- MLP baseline: flatten `28x28` images to `784`, then dense layers.
- CNN baseline: keep `28x28x1` image structure, then use `Conv2D`, pooling, dense classification.

Metrics and outputs:

- Train, validation, and test accuracy for both MLP and CNN.
- Accuracy/loss graph comparing MLP and CNN.
- Saved JSON result summary for the report.

## Part B: Text Task

Use NSMC or the Week10 Korean review data if available locally. If no local review file exists, use the same download/data-loading approach as the professor notebook.

Preprocessing:

- Clean text by keeping Korean, English letters, digits, and spaces.
- Use `KoNLPy/Okt` morphological tokenization if available.
- Fall back to whitespace tokenization when `KoNLPy` is unavailable.
- Convert tokenized text with Keras `TextVectorization`.

Models:

- Required baseline: Bidirectional LSTM with embedding.
- Optional comparison model included: Text CNN with `Conv1D` and global max pooling.

Metrics and outputs:

- Validation and test accuracy for LSTM and Text CNN.
- Text model accuracy/loss graph.
- Saved JSON result summary for the report.

## Report Content

The report will be 2-4 pages in Korean and include:

- Dataset description: MNIST shape, text dataset sentence count, class count.
- Model structures: MLP/CNN for image and LSTM/Text CNN for text.
- Result table: train/validation/test where required.
- Loss decrease graphs for at least one image model and one text model.
- Interpretation:
  - CNN is useful for images because it preserves spatial locality and learns local visual patterns.
  - Sequence models are useful for text because word order changes meaning.
  - The input structure determines whether the model should preserve spatial, sequential, or local n-gram information.

## Verification

Success criteria:

- Text training runs end to end.
- Loss decreases enough to show normal learning.
- Required graphs and result files are generated.
- Notebook opens with the final code/results.
- Report PDF exists and includes the required sections.
